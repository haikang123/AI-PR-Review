# AI PR Review 智能代码评审助手

## 📌 项目简介

本项目为**七牛云实训营大模型应用开发实训项目**，基于 GitHub API \+ 通义千问大模型 \+ Streamlit 搭建的**全自动 AI 代码评审工具**。

项目支持输入任意公开 GitHub PR 链接，自动拉取真实代码变更、智能分析代码问题、输出标准化评审报告，实现轻量化、智能化的 Code Review 能力。

## ✨ 核心功能

- **🔗 真实 PR 解析**：解析标准 GitHub PR 链接，自动获取作者、变更文件、增删代码行数等元数据

- **🌐 在线代码拉取**：调用 GitHub 官方 API，实时拉取真实代码 Diff 变更内容

- **🤖 AI 智能评审**：基于通义千问大模型，自动生成三段式专业评审报告：
        

    - PR 变更总结

    - 风险代码识别（标注高/中/低风险）

    - 可落地优化建议

- **🛡️ 完善异常处理**：适配网络超时、Token 错误、限流、链接格式错误等场景

- **💻 优雅前端交互**：分区渲染、无重复刷屏、状态提示清晰、支持重新评审

## 🛠️ 技术栈

- **后端核心**：Python

- **大模型服务**：阿里通义千问 qwen\-turbo

- **第三方API**：GitHub REST API

- **前端页面**：Streamlit

## 🧠 系统设计思路

#### 1. 模型选择
- **本项目选择阿里通义千问 qwen-turbo 作为核心大模型，理由如下**：
- **低延迟**：响应速度快，平均评审时间在 3-5 秒，满足实时交互需求
- **高性价比**：免费额度充足，适合实训项目和个人使用
- **代码理解能力强**：对 Python 等主流编程语言的语法和最佳实践有良好的理解
- **输出可控性好**：能够严格遵循 Prompt 指令，输出标准化的评审报告格式
- 
#### 2. 上下文获取方式
- **系统采用GitHub REST API 作为唯一数据源，获取 PR 的完整上下文**：
- **PR 元数据**：获取作者、标题、描述、创建时间、变更文件数、增删行数等基础信息
- **代码 Diff**：获取本次 PR 的完整代码变更内容，包括新增、修改、删除的代码行
- **异常处理**：对 API 调用过程中的网络超时、限流、权限错误等情况进行全面捕获和友好提示
- 
#### 3. 误报与漏报控制策略
- **Prompt 约束**：明确要求模型 "只基于提供的代码 Diff 进行分析，不做任何没有依据的猜测"
- **分级评审**：将风险分为高 / 中 / 低三个等级，高风险问题优先展示，低风险问题标注为 "建议"
- **过滤无关内容**：自动过滤纯文档修改、注释修改等非代码变更，减少无效评审

#### 4. 未来扩展方向
- **多语言支持**：扩展支持 Java、JavaScript、Go 等更多编程语言的代码评审
- **上下文增强**：获取 PR 的评论、关联 Issue、历史提交等更多上下文信息，提升分析准确性
- **自定义规则**：支持用户上传团队代码规范，让 AI 按照团队标准进行评审
- **批量评审**：支持一次性评审多个 PR，生成汇总报告
- **集成 CI/CD**：集成到 GitHub Actions，实现代码提交后自动评审
