# AI PR Review 智能代码评审助手

## 📌 项目简介

本项目为**七牛云实训营大模型应用开发实训项目**，基于 GitHub API \+ 通义千问大模型 \+ Streamlit 搭建的**全自动 AI 代码评审工具**。

项目支持输入任意公开 GitHub PR 链接，自动拉取真实代码变更、智能分析代码问题、输出标准化评审报告，实现轻量化、智能化的 Code Review 能力。

## ✨ 核心功能

- **🔗 真实 PR 解析**：解析标准 GitHub PR 链接，自动获取作者、变更文件、增删代码行数等元数据

- **🌐 在线代码拉取**：调用 GitHub 官方 API，实时拉取真实代码 Diff 变更内容

- **🤖 AI 智能评审**：基于通义千问大模型，自动生成三段式专业评审报告：
        

    - PR 变更总结

    - 风险代码识别（标注高/中/低风险）

    - 可落地优化建议

- **🛡️ 完善异常处理**：适配网络超时、Token 错误、限流、链接格式错误等场景

- **💻 优雅前端交互**：分区渲染、无重复刷屏、状态提示清晰、支持重新评审

## 🛠️ 技术栈

- **后端核心**：Python

- **大模型服务**：阿里通义千问 qwen\-turbo

- **第三方API**：GitHub REST API

- **前端页面**：Streamlit

- **网络请求**：requests（适配国内长超时、异常捕获）

## 📁 项目结构

```plain

├── app.py          # Streamlit 前端页面 & 交互逻辑
├── agent.py        # 核心业务逻辑（链接解析/PR拉取/AI评审）
├── .env            # 密钥配置文件（私有不上传）
├── .gitignore      # 项目忽略文件配置
└── README.md       # 项目说明文档
    
```

## ⚙️ 环境配置

在项目根目录创建 `\.env` 文件，配置以下密钥：

```env

# 通义千问密钥
DASHSCOPE_API_KEY=你的通义千问KEY

# GitHub 个人Token
GITHUB_TOKEN=你的GitHub_TOKEN
    
```

## 🚀 运行方式

#### 1\. 安装依赖

```bash
pip install -r requirements.txt 
```

#### 2\. 启动项目

```bash
streamlit run app.py
```
